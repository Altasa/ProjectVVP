***Interfaces***

В файле proreceip функция поиска конкретного рецепта, а также извлечение словарей имен из файла receips.dat
Принимает url рецепта - возвращает словарь с рецептом 
Шаблон вызова функции поиска в файле example_fun

В файле rsauce описание словаря рецепта

Для обновления (создания) файла имен и адресов receips.dat запустите файл receips
Файл receips содержит функцию обновления и записи в файл ./Data/receips.dat
Словарей имен подкатегорий, названий и адресов всех рецептов
(файл данных заполняется ~5 минут)

файл secret_info содержит строку TOKEN='токен собственно'

***Issues***

1
Некоторые категории на сайте либо имеют менее 15 рецептов, либо пусты. 
Эти случаи вызывают словарь рецепта UNKNOWN URL, а также блокируют отображение кнопок
Нужно дополнить модуль очистки в файле proreceip, и наверное перенести его в receips
Чтобы производить очистку перед записью словарей, а не перед запуском бота

2 -!!!-
Существует проблема совместной работы многих пользователей, так как
при одновременных запросах пользователи обращаются к одним и тем же переменным адреса открытого рецепта,
и один пользователь перезаписывает данные другого
Нужно сделать словарь, содержащий для каждого чата адрес открытого рецепта
Возможно, придется записывать его в файл, чтобы перезапускать сервер без потери данных пользователей
В словаре ключ - id чата, значение - url адрес / название рецепта
Также нужно сделать получение компонентов рецепта асинхронным
Т.е. шаги и ингредиенты должны получаться отдельными функциями по url адресу

3
Еще нужно в файле cr_btn_menu сделать автоматическую генерацию buttons в соответствии с subcategories
